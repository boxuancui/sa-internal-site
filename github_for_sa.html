Current Architecture

A repository can be either public or private, either personal or organizational.

Current Architecture

Example of repositories:

  • Organizational public: general Strategic Analytics content with no privacy concerns, e.g., SA Team Site, Tier 1 Cases
  • Organizational private: case work that should not be visible to people outside Strategic Analytics, e.g., B2B Pricing Suite
  • Personal public: your personal playground like Lunch Buddy and Text Miner
  • Personal private: anything personal that you don't want people to see.

Getting Started

To get started, download and install git from git-scm. Once installed, there are two ways to access the repositories on GitHub:

  • Linux Bash: Start -> All Programs -> Git -> Git Bash
  • GUI: Start -> All Programs -> Git -> Git GUI

Creating a repository

To create a new repository, log in GitHub and click the + sign from top right corner.

Creating a repository

Choose if you want to create a personal/organizational, public/private repo and fill in the details.

Add -> Commit -> Push (Git Bash)

Once repo is created, you can push your content with either Git Bash,

Add -> Commit -> Push (Git GUI)

or Git GUI. (Click Stage Changed -> Commit -> Push)

Teams under Strategic Analytics

  • Owner: Full admin, read & write access for administrators
  • ca: Full admin, read & write access for customer analytics
  • ga: Full admin, read & write access for geo-analytics
  • everyone: Read access only for everyone

Adding Collaborators

From your repo, you may add/revoke collaborators from Settings -> Collaborators -> Add Team.

Reminders

  • If you create a private repo (either personal or organizational), it is not visible until you add collaborators.
  • Conversely, if you create a public repo (either personal or organizational), it is going to be visible to everyone. Pay close attention to sensitive information!
